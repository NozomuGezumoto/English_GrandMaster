# リリース前 最終チェックリスト

本番ビルド・TestFlight/内部テスト後に、リリース前に**必ず通す**チェック項目です。

---

## 1. 本番接続の確認（アプリ側）

- [ ] **本番ビルド**で起動（`EXPO_PUBLIC_USE_EMULATOR` を**設定しない**状態でビルドしたもの）
- [ ] アプリ起動後、ログに **`[Firebase] Using production Cloud Functions`** が出ていること  
  → 出ていれば `useEmulator: false` で本番 Firebase に接続できている
- [ ] **`[Firebase] Connecting to emulators`** や **`Connected to Firestore emulator`** が出ていないこと  
  → 出ていたら本番ビルドなのにエミュレータに繋がっている（要修正）

---

## 2. Firebase Console の確認

- [ ] **Authentication**: メール/パスワードが有効。必要なら「メールリンク」等も確認
- [ ] **Firestore**: ルールが「本番に公開」済み。インデックスが「有効」
- [ ] **Functions**: 一覧に Callable が並んでいる。エラー・障害なし
- [ ] **使用量**: 本番テスト後に Firestore 読み書き・Functions 呼び出しが増えていること

---

## 3. 本番ルール・動作の確認

- [ ] **認証**: 新規登録 → ログイン → プロフィール作成ができる
- [ ] **プロフィール編集**: 表示名・国・アバターを変更して保存できる（users の許可フィールドのみ更新される）
- [ ] **ランクマッチ**: マッチング → 10問 → 勝敗 → 結果画面でレート変動が表示される
- [ ] **プロフィール戦績**: 完了したマッチが履歴に表示される
- [ ] **友達**: フレンドコード・リクエスト送信・承認/拒否ができる

---

## 4. 実機・本番特有の確認（見落とし多い）

- [ ] **iPhone 実機**で、リスニング・ディクテーションの**音声が再生される**こと（初回はタップ等のユーザー操作後の再生になる想定）
- [ ] **バックグラウンド・復帰**で、対戦が想定どおり継続または不戦敗になること（必要なら 1 回は試す）
- [ ] **通信が不安定な環境**で、エラー表示やリトライが妥当であること（任意）

---

## 5. リリース直前の確認

- [ ] `app.json` の `extra.firebase.projectId` が**本番用プロジェクト**になっている
- [ ] EAS 等でビルドする場合、本番用プロファイルで **`EXPO_PUBLIC_USE_EMULATOR` を設定していない**
- [ ] ストア申請用のアイコン・スクリーンショット・説明文・プライバシーポリシーが用意できている（ストア提出する場合）
