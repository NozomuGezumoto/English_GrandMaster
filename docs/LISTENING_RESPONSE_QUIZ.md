# リスニング応答選択クイズ — 実装検討

## 1. 概要・要件

- **形式**: 短い英文の音声を聞く → **適切な応答**を4つの選択肢から1つ選ぶ
- **例**:
  - 音声: "It's nice weather today."
  - 選択肢: 無関係3つ + 正解1つ（例: "Let's play outside today."）
- **問題数**: レベルあたり約100問
- **出題元**: コーパスベースで信頼性を確保したい

---

## 2. 実装可否

**結論: 実装可能です。**

既存の仕組みを流用しつつ、新しい問題タイプと再生フローを足せば実現できます。

| 項目 | 現状 | 必要な追加 |
|------|------|------------|
| 問題データ | Firestore `questions` / ローカルJSON（level, prompt, choices, answerIndex） | 新タイプ `listening_response` 用スキーマとデータ |
| 音声再生 | ディクテーションで Expo Speech 使用 | 同じ TTS で「発話文」を再生 |
| 4択UI | 対戦の choice 問題で実装済み | リスニング用は「音声のみ→選択肢のみ」表示に変更 |
| レベル | level 1–10 / TOEIC 帯で出題範囲あり | リスニング問題の level を同じ体系で利用 |

---

## 3. データ設計

### 3.1 問題タイプ

既存の `type`: `cloze` | `reading` に、**`listening_response`** を追加する形が扱いやすいです。

- **prompt**: 音声で流す英文（TTS の元テキスト）。必要なら回答後に画面にも表示可能。
- **choices**: 4つの応答文（正解1 + 誤答3）。表示順はシャッフル可。
- **answerIndex**: 正解の index（0–3）。
- **explanation**: 任意。正解の理由や表現の説明。

（既存の `Question` の `prompt` / `choices` / `answerIndex` をそのまま使えます。）

### 3.2 Firestore との関係

- **方式A**: Firestore `questions` に `type: 'listening_response'` で投入  
  - 既存の出題・レベルフィルタ（`level`, `active`, `qualityStatus`）を流用可能。
- **方式B**: ローカルJSON（例: `data/listening-response.json`）で管理し、アプリまたはスクリプトで Firestore に投入  
  - コーパスから生成した問題をバージョン管理しやすい。

どちらでも実装可能です。まずは方式BでJSONを用意し、必要ならスクリプトで Firestore に一括投入する形が、コーパス運用と相性が良いです。

### 3.3 レベルと問題数

- レベルは既存と同じ **level 1–10**（TOEIC 帯と対応）でよい。
- 「レベルあたり約100問」→ 全レベルで約 1,000 問。
- コーパスから「発話文 + 適切な応答」のペアをレベル別に選び、誤答3つを生成する形が現実的です。

---

## 4. 音声（コーパスから問題を作る前提）

- コーパスは**テキスト**で持つ想定で問題なし。
- **音声は TTS で賄う**（Expo Speech / Web Speech API）:
  - `prompt` をそのまま読み上げ用テキストにする。
  - 録音ファイルを用意しなくても、コーパス由来の英文だけで「リスニング」クイズとして成立する。
- 将来的に「信頼性をさらに上げたい」場合は、重要文にだけ録音を用意し、`audioUrl` があれば再生、なければ TTS にフォールバックする拡張が可能。

---

## 5. コーパス・問題の作り方（100問/レベル）

「コープラスから問題を選ぶと信頼性がある」を満たすための案です。

### 5.1 コーパス候補

- **対話・発話コーパス**
  - 教科書対話、ESL 用スキット、映画・ドラマの字幕などから「短い発話 → その直後の応答」をペアで抽出。
- **既存コーパス**
  - 現在の `corpus-questions.json` は空欄補充（cloze）用。  
    同じコーパスから「全文文」を抜き出し、「適切な応答」を別ルール or 小さい対話データで付与する、という使い方も可能。
- **公開データ**
  - 例: タスク志向対話データセット、映画字幕の隣接発話ペアなど。  
    利用許諾を確認した上で「発話文 → 正解応答」のリストを作成。

### 5.2 問題生成の流れ（案）

1. **(発話, 応答) ペアのリストを用意**
   - コーパスから抽出、またはルール/テンプレートで生成。
   - レベルは語彙・文長・構文で 1–10 を付与（既存の level 体系に合わせる）。
2. **各ペアについて 1 問を作成**
   - `prompt` = 発話文（音声用テキスト）
   - `choices[answerIndex]` = その応答（正解）
   - 誤答3つ: 他問題の応答からランダム、またはテンプレートで生成（レベルが近いものを選ぶと難易度が揃う）。
3. **シャッフル**
   - 選択肢の並び順をランダムにし、`answerIndex` を更新。
4. **出力**
   - レベル別（例: level 1 用 100 問、level 2 用 100 問…）に JSON で保存。  
    必要なら `add-sample-questions.js` と同様のスクリプトで Firestore に投入。

100問/レベルを手で作るのは負荷が大きいので、**半自動**（コーパス抽出 + 誤答生成スクリプト + 軽い目視チェック）が現実的です。

## 6. 実装の進め方（推奨ステップ)

### Phase 1: データと型の準備

1. **型の拡張**
   - `types/firestore.ts`: `QuestionDocType` に `'listening_response'` を追加（または `Question` に `type?: 'listening_response'` を許容）。
   - 既存の `prompt` / `choices` / `answerIndex` をそのまま利用。
2. **サンプルデータ**
   - `data/listening-response.json` を新規作成（レベル別または配列で 10–20 問程度のサンプル）。
   - 形式: `{ prompt, choices, answerIndex, level, explanation?, lang, exam }`。
3. **読み込み**
   - Study 用の `lib/study-questions.ts` などに「listening_response だけをレベルで取得する」関数を追加（ローカルJSON or Firestore）。

### Phase 2: Study タブに「リスニングクイズ」を追加

1. **入口**
   - Study 画面に「Listening Quiz」または「リスニング（応答選択）」のようなボタン/カードを追加。
2. **画面フロー**
   - レベル選択（既存の TOEIC/CEFR 帯でよい）→ 問題リスト取得 → 1問ずつ出題。
3. **1問の流れ**
   - 画面表示時: **音声のみ**で `prompt` を TTS 再生（Expo Speech）。
   - 選択肢は**テキストのみ**（4つ）表示。prompt は最初は非表示（「音声を聞いて応答を選ぶ」形式）。
   - 回答後: 正誤表示とともに、必要なら `prompt` を表示（「聞いていた文」の確認用）。
4. **既存との共通化**
   - 4択の見た目は既存の choice 問題のスタイルを流用し、`type === 'listening_response'` のときだけ「先に音声再生 → 選択肢のみ表示」にする。

### Phase 3: バックエンド（対戦で使う場合）

- 対戦（AI/友達/ランクマ）でリスニング応答問題を出題したい場合のみ実施。
- **createMatch** の `questionType` に `'listening_response'` を追加。
- **getQuestionForMatch** / 出題ロジックで `type === 'listening_response'` かつ level 範囲内の問題を取得。
- マッチ画面では、既存の「4択」フローに「先に prompt を TTS 再生する」処理を足す。

（まずは Study 専用で完結させ、必要なら Phase 3 を後から追加する形でもよいです。）

### Phase 4: 問題数を増やす（コーパス運用）

- 対話コーパス or 既存コーパスから「発話→応答」ペアを抽出するスクリプトを作成。
- 誤答3つを自動生成（他問題の応答 or テンプレート）。
- レベルごとに約100問になるよう生成し、`listening-response.json` を更新（または Firestore に投入）。

---

## 7. まとめ

| 項目 | 結論 |
|------|------|
| 実装可否 | 可能。既存の 4択・TTS・レベル体系を流用できる。 |
| 問題形式 | `type: 'listening_response'`、prompt=発話文、choices=4応答、answerIndex=正解。 |
| 音声 | コーパスはテキストのみで可。TTS で prompt を再生。 |
| 問題数 | レベルあたり約100問は、コーパス＋半自動生成で現実的。 |
| コーパス | 対話・発話コーパスから「発話→応答」ペアを選び、誤答を生成する方式が信頼性と量のバランスが良い。 |

まずは **Phase 1（型・サンプルデータ）→ Phase 2（Study のリスニングクイズ画面）** から進めると、すぐに「聞いて応答を選ぶ」体験まで確認できます。その後にコーパス用スクリプトと問題数を増やす流れがおすすめです。

次に進めるなら、Phase 1 の型定義とサンプル JSON の具体形から詰めていけます。
